# Red Dot Space — V1 Presentation Visual Guide
*For future slides, decks, and presentations · April 2026*

---

## Brand Direction

Red Dot Space should feel like a premium frontier-infrastructure company, not a neon sci-fi concept brand.

Core idea:
- **Dark space first**: deep navy and near-black foundations
- **Blue for structure**: data, interface, technical framing, motion cues
- **Red for signal**: brand punctuation, CTA emphasis, one key highlight at a time

The visual system should feel:
- calm
- technically credible
- editorial
- premium
- space-native

It should not feel:
- green-led
- cyberpunk
- cluttered
- overly futuristic
- deck-like in a literal way

## Colour Palette

| Role | Hex / Value | Usage |
|---|---|---|
| Background (deep) | `#040811` | Page / slide background |
| Background (soft) | `#0b1322` | Secondary section backgrounds, panels |
| Card surface | `rgba(10, 17, 30, 0.76)` | Cards / content panels |
| Card border | `rgba(163, 189, 255, 0.14)` | Subtle outlines |
| Brand Red | `#d42128` | CTA buttons, emphasis, brand punctuation |
| Sky Blue | `#7aa6ff` | Eyebrows, data labels, technical UI, diagrams |
| Sky Blue Soft | `rgba(122, 166, 255, 0.14)` | Soft panel tint, supporting UI |
| Text (primary) | `#f6f3ee` | Headlines and key copy |
| Text (muted) | `rgba(214, 225, 245, 0.70)` | Supporting copy, captions |

### Usage Ratio

- `75%` deep navy / near-black
- `20%` white / muted text
- `5%` accents

Within accents:
- blue should dominate structure and technical framing
- red should be restrained and intentional

### Rules Of Thumb

- Use **one red focal point per visual** where possible.
- Blue should carry charts, dividers, labels, and system linework.
- Do not use green as the primary highlight color.
- Avoid turning every important word red.

| Role | Font | Weight | Notes |
|---|---|---|---|
| Wordmark | Space Grotesk | 700 Bold | Uppercase, tracked, stable |
| Hero / display headings | Space Grotesk | 700 Bold | Tight letter-spacing (`-0.04em`) |
| Section headings | Space Grotesk | 700 Bold | Same family, less branded treatment |
| Eyebrows / labels | Space Grotesk | 500 | Uppercase, `0.18em`–`0.2em` tracking |
| Body copy | Inter | 400 Regular | ~`1.7` line-height |
| Metrics / numbers | Space Grotesk | 700 Bold | Used for key callouts only |

### Wordmark Guidance

For v1, the site is **wordmark-first**:
- `RED DOT SPACE`
- no symbol required
- same display family as the site is acceptable for now
- treatment should still feel more controlled than general headings

Recommended treatment:
- uppercase
- consistent tracking
- no decorative effects
- no green glow

Google Fonts:
- Space Grotesk: https://fonts.google.com/specimen/Space+Grotesk
- Inter: https://fonts.google.com/specimen/Inter

## Visual Principles

1. **Dark space first**
- Prefer deep navy and near-black foundations over literal black or green-glow sci-fi.

2. **Editorial hierarchy over decoration**
- Large statements first
- supporting proof second
- diagrams / cards only where needed

3. **Restraint beats novelty**
- premium spacing and disciplined emphasis matter more than more effects

4. **Blue carries structure, red carries signal**
- Blue for labels, diagrams, chart structure, and technical framing
- Red for CTA, punctuation, or one highlighted point at a time

---

## Motion Principles

For presentations, motion should remain minimal and optional.

Use:
- subtle slide transitions only if they help reading flow
- calm, low-amplitude visual movement if animation is used

Avoid:
- fast spins
- decorative gimmicks
- motion that competes with content

---

## Charts & Data Visualization

Charts should be simplified for communication, not maximized for density.

### Rules

- Blue for structure and default data
- White for labels and primary reading
- Red for **one** emphasized datapoint or curve only
- Use fewer chart elements, not more
- Prefer strong summary numbers over dense mini-charts when possible

### Avoid

- green highlight bars / lines
- multiple accent colors in one chart
- slide-deck density copied directly onto web
- over-labeled diagrams that require zooming

---

## Diagrams & System Visuals

Use:
- blue linework
- dark panels
- white labels
- one red emphasis where needed
- layered architecture visuals

Best-fit diagram types:
- stack diagrams
- control-layer diagrams
- orbit / network relationship diagrams

Avoid:
- glowing green boxes
- overcomplicated maps
- too many disconnected icons

---

## Imagery Rules

Preferred imagery:
- earth horizon
- orbital rings
- deep-atmosphere gradients
- satellite / infrastructure visuals that feel engineered, not playful
- subtle starfield / grid textures

Avoid:
- clipart rockets
- generic AI imagery
- heavy neon glows
- overly cinematic scenes that overpower the copy

## Slide Layout Rules

### Cover / title slide
- Background: `#040811`
- Eyebrow: Space Grotesk uppercase, `#7aa6ff`
- Title: large white Space Grotesk
- One red accent only
- Optional: abstract orbital linework or earth-horizon visual, but not required

### Section divider slide
- Full dark background
- Small chapter number or label in blue
- Large white title

### Data / stat slide
- One dominant number
- Supporting line below
- Red only for the single most important callout

### Card / list slide
- Dark translucent panels
- subtle blue border
- Space Grotesk headings
- Inter body copy
- red bullet dots only if necessary

### Architecture / system slide
- Blue lines and nodes
- White labels
- One red emphasis if needed
- Avoid green system highlighting

---

## Tone & Copy Rules

- Use confident, precise language.
- Avoid vague or speculative phrasing.
- Spell out **Neocloud Provider** on first use.
- Keep `Compute in orbit.` with the period.
- Write `Red Dot Space` as three words, never `RedDotSpace`.

---

## Do / Don't

### Do
- Use blue for labels, diagrams, and technical framing
- Use red sparingly for emphasis and brand punctuation
- Keep layouts spacious
- Let headlines carry weight
- Use dark navy backgrounds instead of pure black where possible

### Don't
- Reintroduce green as the primary accent
- Turn every section into a grid of identical cards
- Use more than one red highlight per chart or visual
- Overuse glow, blur, or motion
- Over-design slides when clarity is the main goal
